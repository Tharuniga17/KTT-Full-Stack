XSS --> Cross Site Scripting --> where the attackers puts JavaScript code into a website and the browser runs it 

Normal user:
Hello everyone

Website shows:

Hello everyone

Attacker user:
<script>alert("Hacked")</script>

here browser will execute the code and it shows to everyone

types includes stored xss -> script stored in database, everyone sees it
reflected XSS --> script comes from url  runs instantly
DOM XSS --> Frontend javascipt mistake, will run in browser immediately

to avoid that dont use raw html, use text safe output, security middleware like helmet, xss-clean

SQL Injection --> sometimes attackers put sql code into input to break or manipulate database queries like this

const query = "SELECT * FROM users WHERE email = '" + email + "' AND password = '" + password + "'";

here the attackers may use,

SELECT * FROM users WHERE email = '' OR 1=1 --' AND password = 'anything';

here OR 1 = 1 means always true --> database returns all users 

-- -> comments out rest of query  it works because sql query built being using string concatenation 

types of sql injection:
login bypass --> skip authentication 

data extraction -->steal entire database

Data deletion --> drop tables

to avoid that never use string concatenation , use orm because it automatically escapes input
