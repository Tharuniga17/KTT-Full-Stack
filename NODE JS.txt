NODE JS:

it is a runtime environment, will lets you run javascript code outside of the browser which means on server/computer for that it will use v8 javascript engine , it will convert the javascript code into machine code so that computer can understand and execute it.

Node js was created by Ryan Dahl 

before node was built --> servers use java, python, php but these are blocking model 

some problems are, 
handling multiple users at the same time --> if 1000 people open a website means we need 1000 threads
one request = one thread , so lot of cpu usage and wastage of memory, server slows down 

But node js is asynchronous perform multiple tasks at a time and faster handling

when we run the command of node app.js 

--> node will process starts, our file will be loaded into memory 
--> then node will initialize the environment like global objects, internal modules, event loop
--> v8 engine starts which takes your code and checks syntax 

v8 creates call stack(lifo) and heap memory 

--> stores functions in the stack 
--> heap memory stores objects, arrays, 

The stack will handle the execution and heap will handles storage 

event-driven --> when something happens, run this function 

node js is server side --> backend 

it will handle data, apis, files, server

node js apis --> const fs = require('fs')

can read/write files, access os and handle network

event loop --> it will continuously check the call stack, if the call stack is empty it go for next task, it will be running continuously

libuv --> it is a open source support library which will perform asynchronous operations

Node js is a cli means we can run that from terminal 

node --watch app.js --> will automatically restarts when file changes

node opens > called repl --> read, evaluate, print, loop

for debugging --> node --inspect app.js

npm --> node package manager 

npm init

npm install express

NODE FLAGS:

node -v

for checking syntax errors, -> node --check app.js

node js will execute synchronous code first, then microtasks(promises) then macrotasks(settimeout) 


Client Request
   ↓

KEY FEATURES OF NODE JS:

single threaded architecture
non blocking i/o
it is event driven
cross platform

There are different queues/ phases inside the event loop that will decide when your code runs

console.log("Start") --> will runs immediately in call stack

process.nextTick() --> special queue called next tick queue , it will run after sync code and runs before everything else like promises


STREAMS AND BUFFERS:

Modules in Node js:

Modules --> is a separate file that will contains reusable code

instead of writing everything in one file --> splitting into multiple file 
each file = one module

modules are used to organize code, reuse code and avoid confusion 

fs module(file system) 
--> read files, write files and delete files

const fs = require("fs");

fs.readFile("data.txt", "utf8", (err, data) => {
  console.log(data);
});

fs.writeFile("file.txt","hello",(err) => {}); --> write files

fs.appendFile("file.txt","more data", () => {}); --> add file or data

fs.unlink("file.txt", () => {}) --> delete files

fs.mkdir("folder",()=>{}); --> creates new folder

Read directory => fs.readdir("./",(err,files)=> {});

http module:

used to create web server, handle requests

const http = require("http");
http.createServer((req,res) =>{
res.end("Hello");
}).listen(3000);

server.listen(3000);


path module:

used to work with file and directory paths safely across different operating systems like windows, Linux, maxOS

const path = require("path");
console.log(path.join(__dirname, "file.txt"));

__dirname → gives current folder path
"file.txt" → file name
path.join() → safely combines them


/home/user/project/file.txt

common path methods

path.join() --> joins multiple path parts safely

path.resolve() --> it will give absolute path

path.basename() --> returns file name only like file.txt

path.extname() --> gets file extension path.extname("file.txt"); --> .txt 

path.dirname("/home/user/file.txt"); --> it shows only directory --> /home/user
