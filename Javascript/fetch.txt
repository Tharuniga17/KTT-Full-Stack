XMLHttpRequest is a browser built in object used to send http requests, and get data from server, for uploading files and track/ download progress
before fetch() exist, xhr was the main way to talk to servers

xhr = new XMLHttpRequest();
xhr.open("GET", "https://api.com/data");  configuring request

send request -> xhr.send();

events in xhr looks like,

load success
xhr.onload = () =>{
    console.log("Done");
}

xhr.onerror = () =>{
    console.log("Network error");
}

xhr.onprogress = (event) =>{
    console.log(event.loaded);
}

for aporting we use xhr.abort(); stops request 

REsumable file upload => if a file upload stops in the middle due to internet issue or something, you can continue 
from where it stopped instead of starting again 

if u upload a 500 mb file, 50 % uploaded, internet breaks, then resumable upload solution continues from last saved point

it will split file into chunks instead of sending full file at once
File = 100 MB

Chunk 1 → 0–10 MB
Chunk 2 → 10–20 MB
Chunk 3 → 20–30 MB
...

first split file into chunks -> chunksize = 2 * 1024 * 1024
and then send each chunk and then track progress

if upload stops , server remembers last position

when user comes back and will ask the server
GET /upload-status
server replies with uploaded : 6MB
and it will continue from there

Long Polling:
instead of asking the server of asking data, it tells give me when you have something
normal polling -> we need send requests for every 5s
in long polling -> one request stays open until data is ready


Websocket => it is a permanent connection between browser and server
no repeated requests, no waiting like long polling, they can talk anytime

once connection opened, stays open , dont need to send request again
it uses websocket handshake, it is the process of upgrading a normal http connection into a websocked connection

Client sends HTTP request -> Server checks request If server supports WebSocket → accepts -> Connection becomes WebSocket(full duplex)
